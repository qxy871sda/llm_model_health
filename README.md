# LLM 模型健康度/成功率统计插件

该插件用于统计并总结 MaiBot 使用的 LLM 模型的调用成功率与历史使用总量。

> ⚠️ **关于运行机制与私有实现依赖的说明**：
> 由于当前 SDK 暂无直接提供 LLM 失败/重试统计接口或直接的日志查询 API，为了能够统计到失败的模型请求，本插件的“近期运行状态”模块依赖于宿主程序的私有日志实现：
> 1. **目录依赖**：插件通过从 `ctx.paths.data_dir` 向上三级（即假设标准目录结构为 `<宿主根目录>/data/plugins/<插件ID>`）跳出插件沙箱，定位并直读宿主日志目录 `logs/`。
> 2. **文件名与格式依赖**：插件直读名为 `app_*.log.jsonl` 的宿主日志文件。
> 3. **日志文案依赖**：插件通过正则匹配 `"选择请求模型:"` 和 `"尝试失败，切换到下一个模型"` 等宿主内部未公开的日志文案来解析模型请求尝试与失败事件。
> 
> **维护与失效风险**：
> 以上目录布局、文件名格式和日志事件文案均属宿主软件的私有非承诺实现。如果宿主后续版本对上述任何内容进行重构（如更换日志存储路径、更改文件名、调整日志输出格式或更改匹配词），本插件的近期健康度监控功能可能会**静默失效**。
> 一旦未来 SDK 补齐相关统计或日志接口，本插件将第一时间迁移至标准 SDK 能力。

## 功能
- **近期日志监控**：支持分析最近一段时间或最近 N 次的模型请求，展示各模型的调用次数、成功数、失败数及成功率。
- **历史数据统计**：从数据库中汇总所有历史成功请求的调用次数、累计消耗费用及平均响应耗时。
- **管理员权限**：仅指定的管理员 QQ 允许调出统计报告，支持配置多个管理员 QQ，留空将不允许所有人调用。
- **自定义指令触发**：支持自定义调出统计数据的指令（默认为 `/模型健康度`），防止与其他插件指令冲突。
- **自定义统计范围**：支持配置统计范围（按最近分钟数或最近请求次数统计）。
- **合并转发发送**：报告通过合并转发（Forward）消息发送，阅读体验清晰整洁。

## 指令
- `/模型健康度`（默认）：调出当前的 LLM 模型使用与成功率统计报告。
  - *注：若您在配置中修改了指令触发器，请使用您自定义的指令。*

## 配置项
可通过 `config.toml`文件或WebUI设置页面修改以下配置：
- **启用插件**：是否启用此插件。
- **管理员 QQ**：有权限调出统计数据的管理员 QQ 号，多个 QQ 用英文逗号分隔；**留空将不允许所有用户调用**。
- **指令触发器**：自定义调出统计数据的指令（例如 `/模型健康度`），修改后需重启插件生效。
- **统计范围类型**：选择计算范围的维度，可选 `minutes` (按分钟) 或 `count` (按请求次数)。
- **统计范围数值**：配合统计类型使用，代表最近的分钟数或请求次数（默认 `30`）。
- **历史模型汇总**：读取数据库内模型使用情况，不支持显示成功率。自定义获取天数`sdk_statistics_days = 1-365`
